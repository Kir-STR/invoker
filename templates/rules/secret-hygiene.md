# Secret hygiene — гигиена секретов

Секреты (API-ключи, токены, приватные ключи, содержимое `.env*`) не выводятся целиком — ни в транскрипт, ни в логи, ни в тела PR / коммитов / комментариев.

## Не дампить окружение

Не перечислять переменные окружения без имени:

- PowerShell: `Get-ChildItem Env:` / `gci env:` / `$env:*`;
- bash/sh: `env` / `printenv` / `set` / `export -p`;
- широкие regex-дампы по `.env*` (grep / Select-String по всему файлу).

Нужную переменную запрашивать **точечно по имени**; значение в выводе маскировать.

## Не эхоить секреты наружу

- Содержимое `.env*` и приватных ключей не вставлять в транскрипт, логи, тела PR / коммитов / комментариев.
- gitignored-файлы секретов (`.env*`) не цитируются в PR ни на каком SHA — reader их не резолвит. Нужно сослаться — опиши словами или reanchor на публичный непроектный пример.
- Перед вставкой вывода команд проверять, что в нём не просочились токены / ключи; при сомнении — маскировать.
