# AGENTS.md — Codex (предпубликационный адвизор)

> **Текущий канон проекта — спека `docs/specs/2026-06-15-invoker-design.md`** (проектного `CLAUDE.md` пока нет; общая дисциплина — в глобальном `~/.claude/CLAUDE.md`). Этот файл не дублирует канон: читаешь интерактивно — открой спеку для контекста. Ниже — только роль Codex в этом проекте.

## Роль

Codex — **предпубликационный адвизор**, не CI-гейт. Сверяет артефакты **до коммита/публикации**: спеку и её правки, описания PR, шаблоны (`templates/**`), правила (`.claude/rules/*.md`), инструкц-файлы (`CLAUDE.md`/`AGENTS.md`/`GEMINI.md`), реестр `modules.json`. Цель — поймать завышенные утверждения, неверные ссылки и scope-drift до того, как они уйдут в историю репозитория.

## Рубрика сверки

- **Границы MVP.** Артефакт не тащит вне-MVP-сущности в реализацию: runbooks / I-P split, поля реестра `requires`/`conflicts`/`maturity`, авто-перевод инструкций, авто-детект значений проекта — отложены (спека § 2, § 9). Не «реализуем заодно».
- **Агностичность.** Поведенческие правила ссылаются на «твой скилл планирования/worktree (или эквивалент)», не вендорят чужие плагины и не дублируют чужой код (спека § 1).
- **init не синтезирует текст.** `init` копирует шаблоны дословно и собирает инструкц-файл включением/исключением размеченных блоков `<!-- module:<id> -->`; он не генерирует правила и не заполняет `{{placeholders}}`, не задаёт вопросов про значения проекта (спека § 5).
- **retro gated.** Единственное автоизменение ФС у `retro` — сведение кандидатов в `ideas_4_rules.md`. Промоут в правила и cleanup `retro-*.md` — только после явного подтверждения оператора (спека § 6). Никакого автономного самопереписывания инструкций.
- **Согласованность реестра.** Реестр `modules.json` живёт в плагине и в целевой проект не копируется; в проект пишется слепок `.invoker/modules.json`. `depends_on`-каскад резолвится и показывается оператору (спека § 3).
- **Перекрёстные ссылки.** `§`-якоря, пути файлов, имена модулей, номера фаз/PR — существуют и указывают куда заявлено. Не ссылаться на несуществующие файлы.
- **Утверждения ⇄ доказательства.** Каждое фактическое утверждение подкреплено источником (git / файл / спека), не памятью. Статус артефакта точен (напр. «operator-approved, pending implementation plan», а не «approved» без источника).

## Дисциплина

- **Возражение с доказательством** работает в обе стороны: сверяй с первоисточником (git / файл / спека), а не с памятью; если факты против инструкции — возрази с доказательством.
- Итераций ≥ 2 на сложный артефакт — норма, не провал; сходимость (каждый раунд — меньше правок) — здоровый сигнал.
- Имена — ASCII; проза — русская.
